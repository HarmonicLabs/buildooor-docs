# `TxBuilderRunner`

```ts
class TxBuilderRunner
{
    constructor(
        txBuilder: TxBuilder,
        provider: Partial<ITxRunnerProvider>
    )
}
```

The `TxBuilderRunner` is an abstraction over the [`TxBuilder`](../TxBuilder%20API/TxBuilder).

To get `TxBuilderRunner` you will infact need a [`TxBuilder`](../TxBuilder%20API/TxBuilder)
and some `Partial`\<[`ITxRunnerProvider`](./IProvider)\>.

For this reason, while there is nothing wrong in creating a `TxBuilderRunner`
directly calling the constructor;
it is suggested to use the [`runWithProvider` method on a `TxBuilder` instance](../TxBuilder%20API/runWithProvider)
to create one instead.

The [`ITxRunnerProvider`](./IProvider) is `Partial` because
a `TxBuilderRunner` instance will do its best to minimize calls to the provider, so if used properly, it is not needed.

For a more detialed explanation on how the provider works
please see the [`runWithProvider` documentation](../TxBuilder%20API/runWithProvider).
